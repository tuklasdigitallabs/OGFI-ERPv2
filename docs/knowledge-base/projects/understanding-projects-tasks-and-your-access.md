# Understanding Projects, Tasks, And Your Access

**Audience / required role:** Project sponsors, project managers, contributors, viewers, and administrators with project access  
**Applies to:** Phase 1.5 Projects & Implementation Tracker

## Purpose

Use this article to understand what the Projects Tracker is for and why some projects, tasks, or linked ERP records may be hidden.

Projects coordinate implementation work such as rollouts, training, maintenance, audit actions, marketing activations, and branch improvement work. They do not replace controlled purchasing, receiving, inventory, approval, finance, or audit records.

## How Access Works

- A project belongs to a company and has a defined scope.
- Restricted projects require explicit project membership, explicit project-scope assignment, or authorized project administration scope.
- Unrestricted projects still require matching company/location/project scope, ownership, membership, or manage-level permission.
- Project members may be sponsors, managers, contributors, or viewers.
- Task actions depend on project role, assignment, and permissions.
- Linked ERP records still use the source module's authorization.

## Expected Result

- You see only projects your role, scope, and membership allow.
- You can view project health, members, risks, recent activity, tasks, and milestones when authorized.
- You can mutate project work only when the project and task permissions allow it.
- Restricted linked records show only that a linked record exists.
- Project activity records key project, member, task, link, risk, milestone, and lifecycle changes where the workflow supports them.

## Important Controls And Warnings

- Completing a project task does not approve a Purchase Request, issue a PO, receive stock, post inventory, close a PO, or change an approval state.
- Project comments do not replace ERP approval remarks.
- Project links do not copy confidential operational payloads into task records.
- Do not use a project task to bypass source-record workflow controls.
- Project membership never grants authority over linked operational records.
