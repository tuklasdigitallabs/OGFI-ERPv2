# Training Module — Phase I Branch Manager Quick Start

**Audience:** Branch Managers and Assistant Branch Managers  
**Duration:** 45-60 minutes  
**Prerequisites:** Active branch scope, Purchase Request, approval, transfer, wastage, inventory view, dashboard, and notification access as assigned  
**Related knowledge-base articles:** Creating a Purchase Request; Reviewing and approving a Purchase Request; Requesting stock when a branch item is low; Receiving a warehouse transfer; Logging wastage; Understanding the dashboard, My Tasks, and notifications

## Learning objectives

By the end of this module, participants can:

- Confirm the active branch context before acting.
- Create and submit a Purchase Request.
- Review assigned approval decisions without self-approval.
- Request internal stock through transfer before purchasing.
- Receive transferred stock and record discrepancy evidence.
- Use dashboard cards and notifications to open source records.

## Demonstration flow

1. Sign in and switch to the assigned branch.
2. Open `Operations Dashboard` and explain read-only source-record cards.
3. Create a draft Purchase Request and submit it for approval.
4. Open `Approval Inbox` as an assigned approver and review a different user's request.
5. Open `Inventory -> Transfers` and create a transfer request for available warehouse stock.
6. Open a dispatched transfer and receive accepted, rejected, damaged, or short quantities as applicable.
7. Log a wastage report with reason and evidence reference.
8. Review audit history and notifications created by the workflow.

## Practice exercise

Create a branch replenishment request for a low-stock item. If warehouse stock is available, create a Transfer Request. If not, create a Purchase Request. Submit the request and explain what record remains the source of truth.

## Mobile pilot check

During pilot training, repeat the critical branch actions on a phone-sized viewport or assigned mobile device:

- confirm the selected branch context is visible before submitting a request;
- create or review a Purchase Request without layout overlap;
- open an approval or notification and reach the source record;
- receive a transfer or record discrepancy evidence without losing line context;
- log wastage with reason and evidence reference;
- capture the denied or missing-permission state for a role that should not perform the action.

If a mobile action is confusing or cramped, record it in the pilot defect log with device/browser, screenshot, source record ID, and the workflow step.

## Common errors and recovery

- Wrong branch selected: switch the header location before creating or receiving records.
- Missing approval button: verify assignment, scope, and self-approval controls.
- Transfer discrepancy without evidence reference: enter the required reason and evidence reference before receipt posting.
- Treating dashboard card as an action: open the source record before acting.

## Completion check

- Participant can show the correct branch context, submit a request, identify the next approver/status, and explain why stock changes only through posted receiving, transfer, wastage, or adjustment movements.
