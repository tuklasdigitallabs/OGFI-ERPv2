# Marketing Operations — Build Plan

## Prerequisites

- Shared Work Management Engine is stable.
- User/role/scope, activity, attachment, notification, calendar, and board behavior have passed common UAT.
- Existing marketing-related code and documents have been reviewed to avoid overlap.

## Release slices

### Slice 1 — Marketing workspace baseline

- Marketing module navigation
- Marketing calendar
- Campaign container and Kanban board
- User assignment/My Work integration
- Creative work items
- Comments, attachments, activity, basic reminders

### Slice 2 — Promotions and branch rollout

- Promotion records and pipeline
- Explicit brand/branch applicability
- Promotion calendar entries
- Branch rollout tracker
- Required approval/readiness references

### Slice 3 — New-item launch coordination

- New-item launch record
- Cross-functional readiness checklist/linked records
- Launch calendar and controlled status
- Soft/full launch and post-review workflow

### Slice 4 — Later enhancements

- Deeper performance analytics
- Approved integrations with POS, assets, publishing, or media tools
- Advanced budget/financial linkage

## Non-goals for first build

- Social publishing automation
- POS discount auto-configuration
- Media buying platform integration
- Creative AI generation/publishing
- External agency portal
- Full marketing budget/accounting workflow

## Verification

Release one specialization at a time. Do not mix shared-engine schema changes, broad board UI changes, and promotion/new-item workflow changes into one uncontrolled release.
