# Phase III — Build Backlog and Acceptance Criteria

**Status:** Controlled implementation started; production finance/workforce release remains gated by UAT, security, migration, and owner signoff.

## Objective

Add budget discipline, expense-to-payment control, workforce operational workflows, attendance inputs, schedules, approvals and reporting.

## Backlog Structure

Every build item must include:

- User story and business owner
- Role and scope affected
- Workflow reference
- Data entities and API impacts
- Screen specification and responsive behavior
- UI/UX readiness proof: list-first or task-queue-first workspace, real tabs/subworkspace navigation where multiple jobs exist, pagination beyond 10 records, no repeated inline action forms, no long stacked-card workspace, readable light/dark contrast, and desktop/tablet/mobile usability
- Validation and exception paths
- Audit, notification and reporting impact
- Unit, integration, role-permission and UAT acceptance criteria

## Required Epics

- Budget planning and budget-versus-actual monitoring
- Expense requests, petty cash, advances and liquidation
- Payment requests and three-way matching support
- Employee master records and effective-dated assignments
- Leave, overtime, scheduling, manpower planning and attendance imports
- Training, document expiry and manpower requisition controls

## Phase Acceptance Gate

The phase may proceed to UAT only when:

1. All priority workflows pass end-to-end tests.
2. Transactions preserve company, brand, location/project, department, requester, status and audit context.
3. Role and segregation-of-duties rules pass negative tests.
4. Mobile use cases work for relevant branch, warehouse, project or field users.
5. Reports, exports, notifications, exception handling and rollback behavior are verified.
6. Required master data, migration and configuration activities are completed.
7. Release Readiness has verified UAT records covering Phase 3 finance foundation, workforce foundation, and deferred blocker review; production go-live still requires resolving or formally waiving the deferred blockers.

## UI/UX Completion Gate

Functional wiring is not enough for Phase 3 completion. A finance or workforce milestone may be counted as user-facing `Implemented` only when the visible workspace follows the approved Modern SaaS standard:

1. Finance and Workforce overviews are command centers, not long stacks of cards.
2. Accounts Payable, General Ledger, Bank & Cash, Budget Control, Expense Requests, Cash Advances, Petty Cash, Period Close, and Workforce Operations use focused tabs/subworkspaces for distinct jobs.
3. Each tab/subworkspace leads with a list, queue, workbench, or selected-period detail appropriate to the task.
4. More than 10 possible records requires pagination or approved incremental loading.
5. Record-specific actions use a selected-record drawer/detail/action composer, not repeated inline reason/evidence/action forms in every row.
6. Multi-line and evidence-heavy entry uses a transaction sheet, drawer, stepper, or full-page task mode rather than a cramped centered modal.
7. Light/dark theme contrast, button hierarchy, spacing, text margins, disabled states, and no-overlap behavior are verified at common desktop widths, tablet, and mobile.

If a milestone lacks this UI/UX gate, it must be labeled `backend foundation only` or `not client-presentable` until remediated.
