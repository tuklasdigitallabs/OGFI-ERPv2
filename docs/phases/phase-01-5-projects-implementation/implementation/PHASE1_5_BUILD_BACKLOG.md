# Phase 1.5 — Projects & Implementation Build Backlog

## Epic 1 — Foundation and authorization

- Project domain module and migrations
- Company/brand/location/department optional scope references
- Project roles, membership, restricted visibility, service-layer authorization
- Audit/activity event foundation and attachment policy reuse
- Project template model and configuration validation

**Acceptance:** unauthorized users cannot discover restricted projects, tasks, attachments, or source-record link details.

## Epic 2 — Project lifecycle and templates

- Project draft/create/activate/on-hold/complete/cancel/archive flows
- Template drafting/publishing/archive controls
- Project members and sponsor/manager assignment
- Basic project overview health calculations

## Epic 3 — Tasks and collaboration

- Board and list views
- Task create/edit/assign/status/priority/due-date flows
- Checklist, comments, attachments, activity timeline
- Mobile My Tasks and task-detail flows
- Explicit task completion/reopen/cancel rules

## Epic 4 — Risks, blockers, milestones, and notifications

- Risks and blockers with required reason/owner/severity
- Milestone tracking and at-risk state
- Assignment, due-soon, overdue, blocked, and milestone notifications
- Configurable escalation policies per template/project

## Epic 5 — Linked ERP records and reporting

- Permission-checked source-record links
- Safe source record summary component
- Project health, task register, overdue/blocked, milestone, activity, and linked-record follow-up reports
- CSV/XLSX/PDF export according to core reporting rules

## Epic 6 — Quality and release readiness

- Service authorization tests
- Activity/audit and status-transition tests
- Mobile task flows
- UAT evidence, training impact review, knowledge-base backlog, rollout/runbook

## Explicit deferrals

Timeline/Gantt engine, resource capacity planning, public share links, external collaboration, automation-builder, embedded project budgets, and task-driven ERP mutations.
