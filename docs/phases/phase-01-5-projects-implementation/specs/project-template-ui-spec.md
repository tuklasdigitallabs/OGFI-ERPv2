# Project Template UI Specification

## Purpose

Allow authorized administrators to define repeatable implementation patterns while preventing uncontrolled workflow sprawl.

## Required sections

- Template identity and project type
- Scope applicability
- Default project roles and visibility
- Status configuration
- Default task groups/checklists
- Milestones
- Required completion evidence
- Notification/reminder defaults
- Template review/publish/archive state

## Controls

- Draft versus published distinction
- Validation before publish
- Warning that edits affect future projects only
- Template activity history
- Archive rather than delete
