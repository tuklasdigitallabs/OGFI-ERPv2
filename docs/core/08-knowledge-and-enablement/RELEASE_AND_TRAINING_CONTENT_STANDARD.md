# OGFI ERP — User-Facing Release and Training Content Standard

## Release summaries

User-facing release summaries answer:

- What changed?
- Which roles or locations are affected?
- What users need to do differently, if anything?
- Are there known limits, prerequisites, or rollout dates?
- Where can users learn more?

They must not expose internal architecture, security details, credentials, or unconfirmed roadmap claims.

Use `docs/templates/RELEASE_NOTE_TEMPLATE.md`.

## Training materials

Training content supports adoption and does not replace role permissions or formal company policy.

Each training module should include:

- Intended audience
- Learning objective
- Required access and test data
- Demonstration flow
- Practice exercise
- Common errors and recovery steps
- Completion check
- Related knowledge-base articles

Use `docs/templates/TRAINING_MODULE_TEMPLATE.md`.

## Phase I training groups

| Audience | Initial training focus |
|---|---|
| Branch Manager | Requests, approvals, transfer receipt, branch control dashboard |
| Storekeeper | Receiving, transfers, stock counts, wastage, evidence attachments |
| Warehouse staff | Dispatch, transfer confirmation, stock location discipline |
| Purchasing | PR review, supplier records, quotation/PO lifecycle |
| Finance / Accounting | Approval visibility, discrepancy review, reporting scope |
| System Administrator | Users, roles, scopes, master data governance, audit review |
